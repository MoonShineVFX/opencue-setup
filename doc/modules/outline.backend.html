outline.backend package¶

Module contents¶

PyOutline output modules.

PyOutline can be thought of as separate from OpenCue proper – it is just a job specification after all, and could be used in any number of contexts.

To this end, PyOutline supports launching jobs on any number of “backends”, i.e. the system responsible for processing the job – launching the job / frames, storing job state, etc.

The main backend of course is OpenCue (outline.backend.cue), which launches the job on OpenCue. However this can be extended to support any job management system. We also include a “local” backend (outline.backend.local) which just runs the job on the current machine, using a SQLite database for storing state.
